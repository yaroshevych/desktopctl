## Iteration 3 - Grid Noise Reduction Tuning (2026-03-17)

### Goal
Reduce box flood while preserving broad pseudo-label recall.

### Changes
- Tuned `grid_cell_boxes` in `src/desktop/daemon/src/vision/tokenize_boxes.rs`:
  - stricter minimum line counts (`h>=6`, `v>=4`),
  - per-line edge coverage gating (`h>=24%`, `v>=20%`),
  - max candidate cell area cap at `24%` of full image.
- Reverted over-aggressive component/morphology changes from failed sub-iteration to preserve recall.

### Measured result
Command:
- `cargo test -p desktopctld broad_grounding_labels_have_minimum_box_recall -- --nocapture`

Metrics:
- recall `0.519` (was `0.532` baseline)
- precision `0.298` (was `0.286` baseline)
- matched expected: `1624 / 3127`
- predicted boxes: `6489` (was `6918`)
- matched predicted: `1932`

Interpretation:
- Small recall decrease, meaningful precision increase, and fewer total predicted boxes.
- This is a better balance for current pseudo-label objective.

### Claude Review (pre-commit)
Brief findings:
- Changes in `grid_cell_boxes` look directionally good for false-positive suppression.
- No blockers found.
- Noted portability concern in `ask_claude.sh` due hardcoded absolute path (user-owned script change, outside this iteration's detector scope).

### Next iteration focus
- Raise precision floor in test after one more controlled tuning step.
- Add per-image worst-case diagnostics to guide targeted heuristic updates.

## Iteration 6 - Tokenize CLI/Schema Expansion + Determinism and CI Gate (2026-03-17)

### Goal
Implement the remaining practical Phase 3/4 deliverables from `20260317-tokenise-plan.md` after detector metrics stabilized: richer tokenize schema, window-targeted CLI surface, screenshot-mode tokenization, glyph emission, static overlay verification, and release-gate automation.

### Changes
- Protocol + schema (`src/desktop/core/src/protocol.rs`):
  - Expanded `Command::ScreenTokenize` fields:
    - `overlay_out_path`
    - `window_id`
    - `screenshot_path`
  - Expanded `TokenizeWindow` payload with:
    - `app` (optional)
    - `os_bounds` (optional original OS bounds)
- CLI (`src/desktop/cli/src/main.rs`):
  - Added tokenize flags:
    - `--overlay <path>`
    - `--window <id>`
    - `--screenshot <path>`
  - Added parser guard rejecting `--window` + `--screenshot` together.
  - Added parser tests for new tokenize modes and invalid flag combination.
- Daemon command handling (`src/desktop/daemon/src/daemon.rs`):
  - `screen tokenize` now supports:
    - explicit window targeting by ID/query,
    - screenshot-path tokenization,
    - overlay writing from tokenize payload.
  - Added `resolve_tokenize_window_target` helper with frontmost/visible fallback heuristics.
  - Added tests for target resolution behavior.
- Vision pipeline (`src/desktop/daemon/src/vision/pipeline.rs`):
  - Added `TokenizeWindowMeta` and refactored tokenize flow:
    - `tokenize_window(...)` for live window crops,
    - `tokenize_screenshot(...)` for offline image tokenization,
    - shared `tokenize_from_snapshot(...)` path.
  - `build_window_elements(...)` now emits:
    - OCR text elements,
    - Rust box elements,
    - Rust glyph elements.
  - Added stable window metadata propagation (`id`, `title`, `app`, `os_bounds`).
  - Added deterministic test for repeated element generation.
  - Added overlay writer test ensuring PNG emission.
- Glyph detector (`src/desktop/daemon/src/vision/tokenize_boxes.rs`):
  - Added `detect_glyphs(...)` using conservative connected-components flow.
  - Added central constants for thresholds/caps.
  - Added text-overlap and centroid-within-text suppression.
- Corpus regression tests (`src/desktop/daemon/tests/tokenize_box_labels.rs`):
  - Added glyph determinism + overlap-budget test on sampled corpus images.
  - Reused overlap threshold constant from detector module.
- Release/CI gate script (`src/desktop/scripts/dev/tokenize_ci_gate.sh`):
  - Runs `desktop-core`, `desktopctl`, `desktopctld` tests.
  - Runs deterministic + corpus box-recall checks.
  - Enforces host stable app hash unchanged before/after.
  - Optional VM smoke via `VM_SMOKE=1` with `VM_SKIP_HOST_BUILD=1`.
