Desktop Control

Computer Vision and mouse/keyboard control for local AI agents. Bring your own AI, observe UI, and perform complex operations with mouse, keyboard, and GPU-accelerated text recognition.

Privacy-first, free & open source, written in Rust.

View on GitHub Read the spec desktopctl ui click --text "Send"

What is this

  • AI agents already work with local files, system applications, and web services.
  • DesktopCtl brings eyes and hands, which control OS and any UI app.
  • GPU-accelerated text recognition and computer vision, on your local machine.
  • Bring your own AI: integrate with any local agent, using standard CLI interface.

CLI interface

zsh — desktopctl 
# list all open windows
$ desktopctl window list

# bring a window into focus by title
$ desktopctl window focus --title "Messages"

# extract structured information from focused window
$ desktopctl screen tokenize --json

# click a UI element by visible text label
$ desktopctl ui click --text "Send"

# block until expected text appears
$ desktopctl wait --text "Sent" --timeout 3000

How it works

Step 01
⬛

Fast loop: local vision

GPU-accelerated text recognition and Computer Vision systems capture UI and extract structured content.

Step 02
⊞

Slow loop: AI reasoning

Agent reads structured content, runs inference on remote server, and interacts with user.

Step 03
▶

Action loop: UI interaction

DesktopCtl interacts with UI, using mouse and keyboard. Desktop apps perform actions and update UI.

Stay in control

  • Computer Vision and text recognition work locally on your machine.
  • By default, DesktopCtl does not share sensitive screenshots with AI agents.
  • Visual effect, when AI agent is accessing screen, mouse, or keyboard.
  • Stay in control with OS-level permissions, granted to DesktopCtl only.
